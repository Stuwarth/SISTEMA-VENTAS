Todo en el orden mencionado,  con  el  propósito  de  abordar  el  problema  presente  dentro  de  la empresa,  identificado  durante  la  investigación  interna,  y  aprovechando  el conocimiento adquirido sobre lo que se requiere y lo que se podrá implementar adicionalmente para facilitar el manejo administrativo dentro de las instalaciones de la empresa Multifases.

Esto también permite mostrar a los usuarios, especialmente al administrador, cómo se verá el sistema, con el fin de recibir retroalimentación y realizar ajustes si es necesario, cómo agregar funciones adicionales, espacios o actividades dentro del sistema según las sugerencias del administrador de la empresa. 

En el Capítulo 2 se analizaron las metodologías más relevantes y utilizadas en el desarrollo de sistemas, como Cascada y Agile, con el propósito de determinar la más  adecuada  para  el  desarrollo  del  sistema  en  la  Empresa  Multifases  y  la resolución del problema existente. Para este proyecto, se seleccionó la Metodología Cascada. 

Esta metodología fue elegida debido a su enfoque estructurado en etapas, que permite abordar cada fase del ciclo de vida del proyecto de manera secuencial, garantizando que cada etapa se complete correctamente antes de avanzar a la siguiente. ![ref3]

Este enfoque minimiza la posibilidad de arrastrar errores o conflictos de etapas anteriores, garantizando así la entrega de un proyecto con la menor cantidad de problemas  posibles  y  proporcionando  una  solución  efectiva  para  la  empresa. Aplicando la metodología Cascada hacia el proyecto, se ha dividido el proceso en las siguientes etapas: 

- Análisis de requisitos: Esta etapa se recopilan los datos necesarios de la empresa para determinar exactamente qué se necesita. 
- Diseño del sistema: Aquí se realiza el diseño de cómo funcionará el sistema cuando sea utilizado por el usuario, así como el diseño de la base de datos para comprender mejor las tablas y relaciones necesarias para cumplir con las funciones del sistema. 
- Desarrollo  del  Sistema:  Con  el  diseño  terminado  y  aprobado  por  el coordinador  de  la  Empresa  Multifases,  se  procede  a  la  codificación  del sistema propuesto. 
- Pruebas del Sistema: Una vez completada la etapa de desarrollo, se realizará pruebas para identificar posibles fallas, documentarlas y encontrar formas de corregirlas. 

Las  últimas  etapas  de  la  metodología,  despliegue  y  mantenimiento,  no  se consideran en esta propuesta, ya que el sistema se presenta como una solución propuesta. Por lo tanto, estas etapas no se incluyen en el proyecto actual. 

Estas razones justifican la elección de la metodología Cascada para el desarrollo del sistema propuesto, asegurando un enfoque estructurado y claro en cada fase del proyecto. Una vez completadas las dos primeras etapas, se procederá a la última etapa del capítulo: el desarrollo del sistema. Este se realizará de acuerdo con lo mostrado y diseñado en las etapas anteriores, con el objetivo de abordar y solucionar el problema en el que se enfoca el proyecto. ![ref3]

1. **Registros  de  Contratos:**  Los  contratos  actualmente  se  almacenan  en archivos físicos, lo que no resulta óptimo para su manejo. Implementar un sistema  más  organizado  y  eficiente  permitiría  mejorar  la  gestión  de  la información contractual. 
1. **Registros de Personal y Proveedores:** Aunque el administrador lleva a cabo estos registros en su oficina, el uso de documentos físicos implica un riesgo significativo de pérdida de información. Es indispensable adoptar un sistema digital más confiable para el almacenamiento de datos y minimizar estos riesgos. 

En resumen, la gestión de registros y el almacenamiento de datos representan el área más crítica para mejorar. Dado que la información contenida en estos registros es esencial para diversas actividades empresariales, como la atención al cliente y el seguimiento de contratos, optimizar estos procesos permitirá evitar pérdidas y garantizar un funcionamiento más eficiente en todos los aspectos del negocio. ![ref3]

**Tabla: Productos** 

- **Código (PK)**: varchar (20) 

  Identificador único para cada producto dentro del sistema. Este código no puede ser nulo y debe ser único. 

- **Identificador**: varchar (20) 

  Código que hace referencia al producto en el sistema. Se utiliza para realizar la distinción entre productos similares. 

- **Marca**: varchar (50) 

  Representa la marca del producto, como Samsung, Sony, etc. ![ref3]

- **Modelo**: varchar (50) 

  Hace referencia al modelo específico del producto, como "LU57Y897" o "D66787D". 

- **Nombre**: varchar (100) 

  Nombre comercial del producto, por ejemplo, "Televisor LED", "Impresora térmica", etc. 

- **Disponibilidad**: varchar (20) 

  Indica el estado del producto. Los valores posibles son: "Disponible", "Alquilado", "Daño", etc. 

- **Identificador de Proveedor**: varchar (20) 

  Clave foránea que se asocia al proveedor del producto, sirviendo para enlazar la información de proveedores. 

**Tabla: Alquileres** 

- **Código de Alquiler (PK)**: varchar (20) 

  Identificador único para cada transacción de alquiler. 

- **Empresa del cliente:** varchar (40) 

  Empresa del cliente que ha alquilado el producto 

- **Ubicación de la empresa:** varchar (60) 

  Ubicación de la empresa del cliente que alquilo el producto ![ref3]

- **C.I del Cliente**: varchar (10) 

  Carnet de identidad del cliente que ha alquilado el producto. 

- **Nombre del Cliente:** varchar (40) 

  Nombre del cliente que alquilo el producto 

- **Apellido del cliente:** varchar (40) 

  Apellido del cliente que alquilo el producto 

- **Correo del cliente:** varchar (90) 

  Correo electrónico que realizo el alquiler 

- **Detalles del alquiler:** varchar (90) 

  Detalles del contrato de alquiler para realizar una explicación. 

- **Equipo alquilado:** varchar (30) 

  Definir el tipo de producto que se alquilo 

- **Cantidad de productos alquilados:** varchar (30) 

  Cantidad de productos que se alquilaron 

- **Fecha de Inicio**: *date* 

  Fecha en la que comienza el alquiler. 

- **Fecha de Fin**: *date* 

  Fecha en la que finaliza el alquiler. ![ref3]

- **Sucursal de Multifases (FK)**: varchar(20) 

  Relacionado con la tabla de la empresa Multifases. Indica qué sucursal está realizando el alquiler.
